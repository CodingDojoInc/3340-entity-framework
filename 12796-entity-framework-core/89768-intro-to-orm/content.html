Intro to ORMs

Learning Objectives:

  • Summarize what an ORM is and what it does for a full-stack application.

For this course, we will be using MySQL as our database of choice. However, if you have never connected a project to a database before, you may be wondering: how do these two totally separate entities manage to communicate with each other?

There are a variety of ways we can accomplish this communication, and perhaps you have some experience in this already from previous courses or personal experience. The way we will do it here is to use what is known as an Object Relational Mapper (ORM).

What is an ORM?

An Object Relational Mapper acts as a connector between a framework (ASP.NET Core) and a database (MySQL). Many programming languages have frameworks that allow them to use an ORM to connect easily with a database.

The super neat thing about using ORMs is that they will not only make talking to and manipulating data in our databases easy, but they will even build our tables for us! If you have ever had to make your tables from scratch or by forward engineering, you know this can be time-consuming.

Now we will be able to quickly and easily create tables in our database based on the models we have been creating in our projects. We won't have to go into MySQL at all unless we want to!
