Entity Framework Core

Learning Objectives:

  • Summarize Entity Framework Core and its role in full-stack development.
  • Ensure that all pre-requisite installations are complete.

Entity Framework Core (typically shortened to EF Core) is the newest incarnation of Entity Framework. It was designed to be more powerful and flexible than Entity Framework and supports a wider variety of databases.

As with all of the Core technologies that Microsoft has created, EF Core is more modular than its previous versions. This means we can pull in just the pieces we need and avoid extra code bloat.

EF Core is a fully featured ORM that is able to map our model classes to SQL tables and format the exact SQL statements we need from the method calls that we make. Basically, once our models are set up in our project, EF Core does all the work of creating the database and tables within MySQL for us! We also have access to methods that will query our database to access or modify these tables without ever opening MySQL or writing a raw SQL statement!

Used in tandem with LINQ, EF Core will make working with a database a breeze. However, there are some careful steps we need to take to get our project set up to talk to a database. This is a bit of a lengthy process, but it is the same steps every time which will eventually make setting up your connection to MySQL feel easy.

Before we get started, we'll first ensure you have everything you need to start working with Entity Framework Core and MySQL.

At this time, please verify that you have MySQL installed on your machine. If you followed the instructions at the beginning of this course, this should be done, but it's good to double-check. Sometimes issues with the program will pop up while you were away.

Open up MySQL Workbench to ensure that everything is in running order or verify that MySQL is installed however you like. The important thing is that it is there and ready for action.

Also at the beginning of the course, you should have installed the following CLI tool from your terminal:

dotnet tool install --global dotnet-ef

This is a global installation, meaning it only needs to be run on your computer once. If you followed the instructions at the beginning of this course you do not need to run this command again. However, if you skipped over any parts, you may want to check that the tool is in use by typing:

dotnet ef

The output will tell you the version of the tool you are using (and show a neat picture of a unicorn!) We will need this tool going forward.

If everything is installed, you're ready to get started! Head to the next module to begin the setup process.
