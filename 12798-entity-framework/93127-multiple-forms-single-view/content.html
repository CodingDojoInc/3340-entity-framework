Multiple Models Single View

Learning Objectives:

  • Create a single View that is capable of handling multiple Models.

Sometimes we will encounter scenarios in which there are multiple forms on a single View. However, we have a problem: if we want to take full advantage of our ASP features, we can only use one model at a time on any given page. Try writing @model int and @model string on the top of any cshtml file and you'll receive a compiler error that "The 'model' directive may only occur once per document".

So how do we handle using multiple forms on one page?

Partials! One of the really neat things about using partials is that the files you create as partials are technically still their own pages. This means that you can have a model in one partial and a model in another and render both those partials on one page without conflict!

Say for example we have an application where we could add a product and a user on the same page. After building out models for both, you would need to create partials for each form.

_ProductForm.cshtml

@model Product
<form asp-action="CreateProduct" asp-controller="Home" method="post">    
    <div class="mb-3">        
        <label asp-for="ProductName"></label>        
        <input asp-for="ProductName" class="form-control">        
        <span asp-validation-for="ProductName" class="text-danger"></span>    
    </div>    
    <div class="mb-3">        
        <label asp-for="Price"></label>        
        <input asp-for="Price" class="form-control">        
        <span asp-validation-for="Price" class="text-danger"></span>    
    </div>    
    <input type="submit" value="Add Product">
</form>

_UserForm.cshtml

@model User
<form asp-action="CreateUser" asp-controller="Home" method="post">    
    <div class="mb-3">        
        <label asp-for="Username"></label>        
        <input asp-for="Username" class="form-control">        
        <span asp-validation-for="Username" class="text-danger"></span>    
    </div>    
    <div class="mb-3">        
        <label asp-for="Age"></label>        
        <input asp-for="Age" class="form-control">        
        <span asp-validation-for="Age" class="text-danger"></span>    
    </div>    
    <input type="submit" value="Add User">
</form>

Then finally the page that shares these forms would look like this:

Index.cshtml

<h1>Add a Product</h1>
@await Html.PartialAsync("_ProductForm")
<h1>Add a User</h1>
@await Html.PartialAsync("_UserForm")

From here, we can set up our controller to handle both requests as normal. No modifications are needed since both forms trigger their own unique post request and have their unique model identified from the start.
