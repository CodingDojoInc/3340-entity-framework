Lambda Expressions

When using method syntax, we have to learn about lambda expressions. A lambda expression is a way of writing an otherwise cumbersome function in a simplified inline expression.  We will pass a lambda expression as the condition or parameter on which we want the query to execute.  Let's take a look at an example of a lambda expression:

List numsAboveTen = numbers.Where(num => num > 10).ToList();

 .Where is a LINQ method and the => is a lambda function. A lambda function contains our query instructions. These instructions are passed into the LINQ method so LINQ can fetch the specific data we ask for.  

This LINQ query filters a list of numbers into a new list of numbers greater than 10. The program will iterate through the original list, and execute the lambda function foreach item in the list. In this case, it will iterate through a list of integers and add each number that meets the requirements (>10) to the new list.


Parts of a Lambda

A lambda function has two parts, the part to the left of the => is a parameter for the function, and to the right of the => is an expression for querying. The parameter to the left of the => can be named anything since parameters are like variables. Parameters hold the data that was passed in to the function.

LINQ queries iterate over a list and execute the lambda function foreach item in the list, passing the current item into the lambda's parameter so the expression to the right of the => can refer check each item for querying. 

Parameter Naming

In general, parameters should be named after the data they will contain, such as num for a number, or if that number represents a score, score could be more descriptive. However, lambdas are a shorthand function, so shorthand parameter names are often convention. For example, a developer might use n as opposed to num when querying a list of numbers, u when querying a list of User class instances, or tm when querying a list of TextMessage class instances.


Lambda Selector

There are two types of operations our lambda expressions will need to perform: selectors and predicates. A lambda selector is used for certain LINQ methods that only need to be told what property / column to use.

people.Min(person => person.Age);

In this example, the function does the job of selecting the person object with the minimum age out of a List of person objects. 

Lambda Predicate

In grammar, a predicate is one of the main two parts of a sentence. It modifies the subject. A lambda predicate is when the right side of the => evaluates into a boolean, such as using comparison operators or methods that return booleans. Check out this example:

List<Person> namedNeil = persons.Where(person => person.FirstName == "Neil").ToList();

The == equality comparison operator evaluates to a boolean, when your lambda returns a true to .Where, LINQ knows it found something you were looking for. This works for > and < just the same. 

List<Person> nameStartsWithN = persons.Where(p => p.FirstName.StartsWith("N")).ToList();
  • .StartsWith returns a boolean, it is doing a comparison internally.
